<h1>Data Types</h1>

<p>PHP enums are used to specify valid values for properties.</p>
<p>This provides a central location for validation logic and documentation, as well as reducing duplication, ensuring consistency, and enabling type safety.</p>
<div className="two-column-doc-section">
<div>
<h2>Alignment</h2>
<p>Enum used to specify valid values for alignment properties.</p>

<figure>

<figcaption>Supported values</figcaption>

| Case | Value |
|------|-------|
| `START` | `start`|
| `END` | `end`|
| `CENTER` | `center`|
| `JUSTIFY` | `justify`|
| `MATCH_PARENT` | `match-parent`|

</figure>

<figure>

<figcaption>Methods</figcaption>

<dl><dt><code>fromString(string $value)</code></dt><dd>Converts a string to the corresponding enum case.</dd></dl>

<details>
<summary>Supported input mappings</summary>

| Input | Result |
|-------|--------|
| `'start'` | `self::START` |
| `'left'` | `self::START` |
| `'top'` | `self::START` |
| `'end'` | `self::END` |
| `'right'` | `self::END` |
| `'bottom'` | `self::END` |
| `'center'` | `self::CENTER` |
| `'justify'` | `self::JUSTIFY` |
| `default` | `self::MATCH_PARENT` |


</details>
</figure>
</div>
<div>

<figure>

<figcaption>Basic usage</figcaption>

```php
use Doubleedesign\Comet\Core\Alignment;

$result = Alignment::START;
$value = $result->value; // returns 'start'

```

</figure>

<figure>

<figcaption>Method usage</figcaption>

```php
use Doubleedesign\Comet\Core\Alignment;

$result = Alignment::fromString('left');
```


</figure>

</figure>

<figure>

<figcaption>Methods</figcaption>
